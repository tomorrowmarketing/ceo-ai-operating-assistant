# 07. 로드맵

각 단계는 이전 단계 위에 쌓인다. 항상 **읽기 → 요약/우선순위 → 승인 기반 실행** 순서로 확장한다.

## 1단계 — Mock 대시보드 (현재 / MVP) ✅

- Next.js + TS + Tailwind, App Router
- Mock 데이터 기반 일일 운영 브리핑 대시보드 (9개 영역)
- 비즈니스 로직(순수 함수)로 브리핑 파생
- 외부 연동/인증/실행 없음

## 2단계 — 데이터 어댑터 & 실데이터(읽기)

- 도메인 타입을 유지한 채 Mock → 실데이터 어댑터 교체
- 우선 연동(읽기): 캘린더, 메일/메신저(요약), 광고 성과(Meta Ads), 재무(거래 요약)
- 요약/집계만 저장 (민감정보 정책 준수)
- 기준일 `TODAY` 를 실제 날짜로 전환

## 3단계 — 인증 & 접근통제

- 대표 단일 사용자 로그인
- 데이터 암호화/접근통제/감사 로그
- 민감정보 마스킹을 파이프라인 레벨에서 강제

## 4단계 — 승인 기반 실행 (쓰기)

- 승인 큐 활성화: 메일/메시지 초안 발송, 일정 제안 등록 등
- 모든 외부 행동은 대표 승인 후에만 실행
- 실행 결과 로깅 및 되돌리기 고려

## 5단계 — 광고주 360° 뷰 & 보조 예측

- 광고주별 상세 페이지 (계약·업무·커뮤니케이션·일정·재무·성과 통합)
- 이탈 위험/매출 예측 보조 지표
- 주간/월간 리포트 자동 초안

## 비고

- 단계 간 경계는 고정이 아니며, 각 단계는 독립적으로 가치를 준다.
- 과설계 금지 원칙에 따라, 다음 단계 요구가 명확해질 때 구조를 확장한다.
