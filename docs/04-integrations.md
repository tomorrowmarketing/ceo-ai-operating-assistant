# 04. 연동 (Integrations)

## 현재 상태: 연동 없음

1차 MVP는 **Mock 데이터만** 사용한다. 아래 외부 서비스는 **이번 단계에서 연결하지 않는다.**

| 영역 | 대상 | 향후 채울 데이터 |
| --- | --- | --- |
| 문서/지식 | Notion | 광고주 운영 노트, 내부 위키 |
| 메일 | Gmail | 광고주 커뮤니케이션(수신 요약) |
| 캘린더 | Google Calendar | 미팅·상담 일정 |
| 메신저(내부) | Slack | 직원 커뮤니케이션, 내부 알림 |
| 메신저(고객) | 카카오톡 | 광고주 커뮤니케이션 |
| 재무 | 은행 / 카드 | 입금·지출 거래 |
| 세무 | 세금계산서 | 발행/수취 내역 |
| 광고 | Meta Ads | 광고 성과(리드 수, 비용, 전환) |

## 연동 설계 원칙 (향후)

1. **읽기 우선(read-first).** 먼저 데이터를 가져와 요약/우선순위에 쓰고,
   쓰기(외부 행동)는 승인 기반으로 가장 마지막에 추가한다.
2. **요약·집계만 저장.** 원문/개인정보/원본 리드 DB는 저장하지 않는다. (`05-security-policy.md`)
3. **어댑터 패턴.** 각 소스를 `Communication / FinanceTransaction / CalendarEvent` 등
   공통 도메인 타입으로 정규화하는 어댑터를 둔다. 대시보드는 소스를 모른다.
4. **Mock 호환 인터페이스.** 현재 `src/lib/mock/` 가 제공하는 형태와 동일한 인터페이스로
   실데이터 소스를 교체할 수 있게 한다. (대시보드/로직 변경 최소화)

## 쓰기(외부 행동) 연동의 전제

메일 발송, 메시지 발송, 일정 생성/수정, 광고 캠페인 수정 등은
**모두 승인 큐를 거쳐 대표가 명시적으로 승인한 후에만** 실행한다. (`06-agent-rules.md`)
MVP 단계에서는 어떤 쓰기 연동도 하지 않는다.
