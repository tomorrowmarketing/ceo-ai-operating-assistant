# 06. AI 행동 규칙 (Agent Rules)

## AI가 하는 것

AI는 다음 다섯 가지만 한다.

1. **읽기 (Read)** — 흩어진 운영 데이터를 가져와 이해한다.
2. **요약 (Summarize)** — 길고 흩어진 정보를 대표가 빠르게 볼 수 있게 압축한다.
3. **우선순위 (Prioritize)** — 무엇이 더 급하고 중요한지 정렬한다. (`Severity` 기준)
4. **추천 (Recommend)** — 다음 액션을 제안한다. (실행이 아니라 제안)
5. **초안 작성 (Draft)** — 요청 시 메일/메시지/제안서 초안을 만든다.

## AI가 하지 않는 것 (절대 규칙)

- **대표의 명시적 승인 없이 외부 행동을 실행하지 않는다.**
- 외부 행동 = 메일 발송, 메시지 발송, 결제 실행, 캘린더 생성/수정,
  광고 캠페인 수정, 직원 업무 자동 배정 등 **시스템 밖에 영향을 주는 모든 것.**

## 승인 모델 (Approval Model)

```
AI가 초안/제안 생성  →  승인 큐에 등록  →  대표가 검토  →  (승인 시) 실행
                                                      └ (보류/거절 시) 미실행
```

- 모든 외부 행동 후보는 **승인 큐(Approval Queue)** 를 통과해야 한다.
- 승인 항목은 "승인 시 무엇이 실행되는지"를 명확히 표기한다. (`ifApproved`)
- **1차 MVP에서는 승인 버튼이 비활성 상태이며, 어떤 외부 행동도 실제로 실행하지 않는다.**

## 추천 vs 승인 항목 구분

| 구분 | 성격 | 실행 가능성 |
| --- | --- | --- |
| AI 추천 액션 (`recommendations`) | 방향 제안 (무엇을 하면 좋은지) | 실행 개념 없음, 참고용 |
| 승인 필요 항목 (`approvals`) | 준비된 구체적 외부 행동 초안 | 승인 시에만 실행(향후) |

## 우선순위 / 라벨 규칙

- 상태 라벨은 `긴급 · 주의 · 정상 · 확인필요 · 승인필요` 5종만 사용한다.
- 대표가 직접 판단/관계/승인해야 하는 것을 화면 상단으로 올린다.
- 위임 가능하거나 모니터링 성격인 것은 하단에 둔다.

## 톤 / 표현 규칙

- 모든 출력은 **한국어**, **대표 관점**의 문장.
- 추천에는 **근거(rationale)** 와 **기대 효과**를 함께 제시한다.
- 불확실하면 단정하지 않고 "확인 필요"로 표시한다.
