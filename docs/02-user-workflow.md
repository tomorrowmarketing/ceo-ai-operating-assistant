# 02. 사용자 워크플로우

## 주 사용자

**대표 1인.** 광고대행사를 운영하며, 영업·관계·재무·인력을 동시에 챙긴다.
기술 사용자는 아니며, "빠르게 핵심만" 보고 싶어 한다.

## 사업 흐름 (맥락)

```
광고 집행 → 리드폼 제출 → 전화 상담 → 방문/온라인 계약
```

광고주의 성과는 이 깔때기 각 단계의 숫자(리드 수, 상담 전환율, 계약)로 나타난다.

## 대표의 하루 (핵심 시나리오)

### 아침 5분 브리핑 (이 제품의 핵심)
1. 대시보드를 연다.
2. **헤더**에서 오늘의 운영 헤드라인과 핵심 지표(운영 광고주 수, 이탈 위험, 오늘 일정, 지연 업무, 회신 대기, 승인 대기)를 본다.
3. **대표님이 먼저 판단할 항목**(상단)을 보고 오늘 직접 처리할 일을 정한다.
4. **승인 필요 항목**에서 AI가 준비한 초안을 확인한다. (현재 MVP는 승인 실행 안 함)
5. **오늘 일정**으로 미팅/상담을 확인한다.
6. **광고주 리스크 신호 · 중요 커뮤니케이션**으로 이탈 위험과 응대 필요 건을 본다.
7. **직원 업무 병목 · 재무/계약 알림**으로 운영/돈 흐름을 점검한다.
8. **AI 추천 액션**에서 오늘의 기회/대응 제안을 참고한다.

### 결과
대표는 5분 안에 "오늘 무엇을 먼저 판단/결정/위임할지"를 정리한다.

## 정보 우선순위 원칙

화면은 **대표가 먼저 판단해야 할 것 → 광고주 중심 정보 → 운영/재무** 순으로 내려간다.

- 위로 갈수록: 대표의 결정·관계·승인이 필요한 것
- 아래로 갈수록: 위임 가능하거나 모니터링 성격인 것

## 비-목표 사용 흐름

- 직원이 업무를 등록/체크하는 협업 흐름 (이 제품은 대표용)
- 대표가 일일이 질문해서 답을 얻는 챗 흐름 (브리핑이 먼저 떠 있어야 함)
