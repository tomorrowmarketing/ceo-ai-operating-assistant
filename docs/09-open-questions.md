# 09. 미해결 질문 (Open Questions)

구현/제품 판단이 필요하지만 아직 확정되지 않은 항목. 확정되면 `08-decision-log.md` 로 옮긴다.

## 제품

- [ ] 대표가 "직접 판단할 항목"의 자동 선별 기준을 더 정교화해야 하는가? (현재는 긴급 리스크/부정 커뮤니케이션/연체 중심)
- [ ] 일일 브리핑을 특정 시각에 푸시(이메일/메신저)로도 보내야 하는가? (현재는 화면만)
- [ ] 광고주별 상세 페이지(360° 뷰)를 MVP 다음 우선순위로 둘 것인가?
- [ ] "정상" 광고주도 최소한의 긍정 신호(성장/기회)를 보여줘야 하는가?

## 데이터 / 지표

- [ ] 리스크 점수를 규칙 기반 외에 가중치 모델로 바꿀 필요가 있는가?
- [ ] 광고 성과 지표를 어디까지 집계(채널별/캠페인별)로 보여줄 것인가?
- [ ] 미수금/연체 판단 기준(연체 일수 임계값)을 회사 정책에 맞춰 확정 필요.
- [ ] 업종별(병의원/법무/렌트)로 봐야 할 핵심 지표가 다른가? (전환 정의 차이)

## 연동 (2단계 이후)

- [ ] 카카오톡/메일 연동 시 요약 생성을 어느 시점에 수행할 것인가? (수신 시 vs 조회 시)
- [ ] 캘린더 개인 일정의 `isPrivate` 판단을 무엇으로 자동화할 것인가?
- [ ] 광고 성과 소스(Meta Ads)와 내부 계약/재무를 어떻게 매칭할 것인가?

## 보안 / 승인

- [ ] 승인 실행(4단계) 시 되돌리기(undo)와 실행 로그 보관 기간 정책.
- [ ] 단일 대표 사용자 외에 보조 인력 접근을 허용할 것인가? (역할/권한 모델)

## UX

- [ ] 모바일 우선 레이아웃이 필요한가? (대표가 폰으로 아침 확인 가능성)
- [ ] 항목별 "읽음/처리함" 상태를 둘 것인가, 매일 새로 생성할 것인가?
