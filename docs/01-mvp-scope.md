# 01. MVP 범위

## 1차 MVP 목표

**Mock 데이터 기반 웹 대시보드**를 만들어, 대표가 **매일 오전 5분 안에 회사 운영 상황을 파악**하게 한다.

## 포함 (In scope)

### 기술
- Next.js (App Router) + TypeScript + Tailwind CSS
- 데이터는 **Mock TypeScript/JSON만** 사용

### 대시보드 영역 (컴포넌트)
1. 일일 브리핑 (헤더 + 핵심 지표)
2. 오늘 일정
3. 대표가 먼저 판단할 항목 (TOP 액션)
4. 중요 커뮤니케이션
5. 광고주 리스크 신호
6. 직원 업무 병목
7. 재무·계약 알림
8. AI 추천 액션
9. 승인 필요 항목

### Mock 데이터
광고주 · 직원 · 업무 · 커뮤니케이션 · 캘린더 일정 · 재무 거래 · 계약 · 일일 브리핑.

### 비즈니스 로직 (순수 함수)
오늘 일정 요약 · TOP 액션 아이템 · 광고주 리스크 신호 · 직원별 지연 업무 요약 ·
중요 커뮤니케이션 요약 · 재무/계약 알림 · 승인 필요 항목 · AI 추천 액션 · 일일 브리핑 조립.

## 비포함 (Out of scope — 이번 단계에서 하지 않음)

### 외부 연동 안 함
Notion, Gmail, Google Calendar, Slack, 카카오톡, 은행, 카드, 세금계산서, Meta Ads.

### 기능 안 만듦
- 인증/로그인
- 메일 자동 발송
- 결제 실행
- 캘린더 수정
- 광고 캠페인 수정
- 직원 업무 자동 배정

승인 버튼 등 "행동" UI는 **표시만 하고 실제 실행하지 않는다.**

## 완료 정의 (Definition of Done)

- [x] `npm install && npm run dev` 로 로컬에서 대시보드가 뜬다.
- [x] `npm run build` 가 타입 에러 없이 통과한다.
- [x] 9개 영역이 Mock 데이터 기반으로 한국어로 렌더된다.
- [x] 대표가 먼저 판단할 항목이 상단에 온다.
- [x] 민감정보 규칙(`05-security-policy.md`)을 Mock 데이터가 위반하지 않는다.

## 설계 원칙

과설계하지 않는다. 불필요한 의존성을 추가하지 않는다. 첫 버전은 단순하고 리뷰 가능하게.
