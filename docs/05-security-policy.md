# 05. 보안 · 민감정보 정책

이 제품은 병의원·법무법인·렌트/리스 광고주의 데이터를 다루므로 민감정보 취급이 핵심이다.
아래 규칙은 **반드시 준수**한다. Mock 데이터에도 동일하게 적용한다.

## 저장 금지 (Do NOT store)

- **전체 휴대폰 번호** — 마스킹 형태만 (`010-****-1234`)
- **전체 계좌번호 / 카드번호**
- **주민등록번호**
- **환자 상세정보** (진단·진료 내용 등) — 업종 단위 집계만
- **법률 사건 상세정보** (사건 내용·당사자 등) — 업종 단위 집계만
- **원본 리드 DB** (개인 리드의 이름·연락처 등) — 집계 지표(리드 수, 전환율)만
- **개인/가족 일정 상세** — `isPrivate` 처리, 제목·내용 비노출

## 마스킹 규칙

| 항목 | 저장 방식 |
| --- | --- |
| 전화번호 | `010-****-1234` (가운데 마스킹) |
| 이메일(발신자) | 표시명 + 마스킹 도메인 (`masked@clinic.example`) |
| 금액 | 저장 가능(원). 단 계좌/카드 식별정보는 저장 금지 |
| 개인 일정 | 제목을 "개인 일정"으로 치환, 세부내용 미저장 |

## 데이터 최소수집 원칙

- 운영 판단에 필요한 **요약·집계·상태**만 저장한다.
- 커뮤니케이션은 **원문이 아니라 AI 요약**만 저장한다.
- 광고 성과는 **개인 리드가 아니라 집계 지표**만 저장한다.

## AI 행동 관련 보안

- AI는 대표의 **명시적 승인 없이 외부로 어떤 데이터도 전송하거나 행동을 실행하지 않는다.**
- 외부 행동은 승인 큐를 통해서만 가능하다. (`06-agent-rules.md`)

## MVP 단계의 적용

- 인증 기능은 만들지 않지만(범위 외), Mock 데이터는 위 규칙을 **이미 준수**한다.
  - 예: 커뮤니케이션 발신자는 `한결법률사무소 대표 (010-****-3271)` 처럼 마스킹.
  - 예: 재무 거래에는 계좌/카드번호 없이 금액·상태·기한만.
- 실데이터 연동 단계에서 접근통제·암호화·감사로그를 별도 설계한다. (`07-roadmap.md`)

## 점검 체크리스트 (데이터/코드 추가 시)

- [ ] 전체 전화번호/계좌/카드/주민번호가 들어있지 않은가?
- [ ] 환자/사건 상세, 원본 리드 식별정보가 들어있지 않은가?
- [ ] 개인 일정이 `isPrivate` 로 가려지는가?
- [ ] 커뮤니케이션은 원문이 아닌 요약인가?
