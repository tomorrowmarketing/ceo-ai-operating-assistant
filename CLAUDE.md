# CLAUDE.md

이 문서는 AI 코딩 도구(Claude Code 등)가 이 저장소에서 작업할 때 따르는 **단일 기준(source of truth)** 이다.
구현 판단이 애매할 때는 항상 이 문서를 기준으로 결정한다.

---

## 1. 제품 한 줄 정의

대한민국 **리드수집 광고대행사 대표**를 위한 **개인용 AI 운영 비서**.
일반 챗봇이 아니라, 대표가 **매일 오전 5분 안에 회사 운영 상황을 파악**하는 **운영 브리핑 대시보드**다.

- 주 사용자: **대표 1인**
- 주요 광고주 업종: **병의원 · 법무법인 · 장기렌트/리스**
- 사업 흐름: `광고 집행 → 리드폼 제출 → 전화 상담 → 방문/온라인 계약`

## 2. 핵심 업무 원칙

> **광고주 = 계약 + 업무 + 커뮤니케이션 + 일정 + 재무 + 광고 성과**

모든 화면과 데이터는 **광고주 중심**으로 정리한다. 일반 TODO 앱처럼 보이면 안 된다.

## 3. AI 행동 원칙

AI는 다음만 한다: **읽기 · 요약 · 우선순위 결정 · 추천 · (요청 시) 초안 작성.**

- AI는 대표의 **명시적 승인 없이 외부 행동을 실행하지 않는다.**
- 외부 행동(메일 발송, 메시지 발송, 결제, 캘린더 수정, 광고 캠페인 수정, 직원 업무 자동 배정)은
  **승인 큐(Approval Queue)** 를 통해 대표가 승인해야만 실행될 수 있다.
- 현재(1차 MVP) 단계에서는 **어떤 외부 행동도 실제로 실행하지 않는다.** 승인 버튼은 비활성 상태다.

## 4. 1차 MVP 범위

**목표:** Mock 데이터 기반 웹 대시보드로 대표가 매일 오전 5분 안에 운영 상황을 파악한다.

### 기술 스택
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- 데이터는 **Mock TypeScript/JSON만** 사용

### 이번 단계에서 하지 않는 것 (명시적 비범위)
- 실제 외부 API 연동 안 함: Notion, Gmail, Google Calendar, Slack, 카카오톡, 은행, 카드, 세금계산서, Meta Ads
- 인증 기능 없음
- 메일 자동 발송 / 결제 실행 / 캘린더 수정 / 광고 캠페인 수정 / 직원 업무 자동 배정 없음

## 5. 디렉터리 구조

```
src/
  app/                 # App Router (layout.tsx, page.tsx=대시보드, globals.css)
  components/          # 대시보드 UI 컴포넌트 (영역별 1개)
    ui/                # 공통 프리미티브 (SeverityBadge, Section 등)
  lib/
    types.ts           # 도메인 타입 (단일 정의처)
    mock/              # 가상 데이터 (advertisers, staff, tasks, ...) + TODAY 기준일
    briefing/          # 비즈니스 로직 (브리핑 파생 함수) + composeDailyBriefing
docs/                  # 제품 문서 00~09
```

## 6. 데이터 / 로직 규칙

- **타입은 `src/lib/types.ts` 한 곳에서만 정의**한다. 컴포넌트/로직은 여기서 import 한다.
- 비즈니스 로직은 `src/lib/briefing/` 의 순수 함수로 작성하며, **`DataSource` 를 인자로 받는다**(DI).
  Mock 데이터를 직접 import 하지 않는다. UI에 로직을 넣지 않는다.
- 데이터 접근은 **`src/lib/data` 의 `loadDataSource()`** 만 사용한다. (`@/lib/mock` 직접 참조 금지)
  - `DATA_SOURCE=mock|notion` 으로 소스 전환. 기본 Mock, 실패 시 Mock 폴백. (docs/04, docs/10)
  - 새 실데이터 소스는 `DataSource` 를 만족하는 어댑터로 추가한다.
- 날짜 기준은 Mock의 `TODAY`(또는 소스의 `today`)를 사용해 **결정적(deterministic)** 으로 유지한다.
- 운영 상태 라벨은 다음 5종만 사용한다: **긴급 · 주의 · 정상 · 확인필요 · 승인필요** (`Severity` 타입).

## 7. UI 규칙

- **모든 UI 문구는 한국어.** 대표 관점의 문장으로 작성한다. (예: "대표님이 먼저 판단할 항목")
- 깔끔한 비즈니스 대시보드 스타일. **정보 명확성 우선.**
- **대표가 먼저 판단해야 할 항목을 상단에 배치**한다.
- 데이터는 광고주 중심으로 묶어 보여준다.

## 8. 민감정보 규칙 (반드시 준수)

다음 정보는 **저장하지 않거나 마스킹**한다.
- 전체 휴대폰 번호 → `010-****-1234` 형태로만
- 전체 계좌번호 / 카드번호 / 주민등록번호 → 저장 금지
- 환자 상세정보 / 법률 사건 상세정보 → 저장 금지 (업종 단위 통계만)
- 원본 리드 DB → 저장 금지 (집계 지표만: 리드 수, 전환율 등)
- 개인/가족 일정 상세 → `isPrivate` 처리, 제목·내용 비노출

Mock 데이터를 만들 때도 이 규칙을 그대로 따른다.

## 9. 작업 원칙

- **과설계하지 않는다.** 불필요한 의존성을 추가하지 않는다.
- 첫 버전은 **단순하고 리뷰 가능**하게 만든다.
- 새 기능/결정은 `docs/08-decision-log.md` 에 한 줄 기록한다.
- 미해결 질문은 `docs/09-open-questions.md` 에 적는다.

## 10. 실행

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # 프로덕션 빌드 검증
```

상세 문서는 `docs/` 참고. 다른 AI 도구는 `AGENTS.md`(이 문서의 호환 사본 안내)를 본다.
